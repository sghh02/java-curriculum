# 環境構築


ここからは実際にJavaのプログラムを動かしながら、プログラムの書き方を勉強していきます。
まず、カリキュラムを行うのに必要な環境構築をしてもらいます。
これは、実際の開発現場でも最初に行う作業です。
Javaを動かす環境と、課題を提出できる環境を作成してみましょう。

## 環境構築とは

・環境を作る作業のこと
・「環境」が何かはケース・バイ・ケースだよ
・実際にやる作業はケース・バイ・ケースだよ

今回行う環境構築は以下になります。
・Javaでプログラミングを行うための環境構築

## Javaでプログラミングを行うための環境構築

## Java（JDK）の準備（推奨：17）

このカリキュラムは最終的に Spring Boot（Unit 9）まで進みます。
Spring Boot 3系は **Java 17** が前提になることが多いので、JDK 17 を推奨します。

### 確認方法

ターミナル（またはコマンドプロンプト）で次を実行して、バージョンを確認してください。

```bash
java -version
```

表示が古い/`java` が見つからない場合は、AIに次のように質問してOKです。

```text
OSは（Windows/macOS）です。
`java -version` を実行すると（ここに結果）になります。
Java 17 を使える状態にしたいので、必要な手順を教えてください。
```

## IDE のインストール

**IDE（統合開発環境）** は、コードを書く/実行する/デバッグするための道具です。
このカリキュラムでは例として **Eclipse** を想定しますが、IntelliJ IDEA / VS Code などでもOKです。

### Eclipse（例）

- 日本語版をまとめて入れたい場合：Pleiades を使う
  - https://willbrains.jp/

インストール中に警告やブロックが出たら、画面の文言をそのままAIに貼って「安全に進める手順」を確認してください。

### IntelliJ IDEA / VS Code（任意）

Spring Bootは、IntelliJ IDEA Community や VS Code でも開発できます。
使い慣れたものを選んでOKです。

## Git の準備（推奨）

制作物の提出や、Unit 10（Petclinic）で必要になります。

```bash
git --version
```

`git` が見つからない場合は、OS名と状況をAIに貼って手順を確認してください。

```text
OSは（Windows/macOS）です。
`git --version` を実行すると（ここに結果）になります。
GitHubにpushできる状態にしたいので、必要な手順を教えてください。
```

## （任意）Eclipseの補助設定

コード補完（コンテンツアシスト）は、好みで調整してOKです。
設定場所が分からない場合は「Eclipse コンテンツアシスト 自動有効化」などで調べるか、AIに聞いてください。
