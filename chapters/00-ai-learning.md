# AI時代のJava学習法

このカリキュラムを始める前に、**AI時代の効率的な学習法**を身につけましょう。
従来の「教科書を読んで暗記する」学習は、もはや最速ではありません。

---

## なぜAIを使った学習が最速なのか

プログラミング学習の最大の壁は「わからないときに止まること」です。

- 従来：わからない → 検索 → 記事を読む → それでもわからない → 諦める
- AI活用：わからない → AIに聞く → すぐ答えが返る → 続けられる

**続けられることが最重要**です。AIを使えば「止まる時間」を最小化できます。

---

## Step 1：止まらずに進めるためにAIに聞く

最初から完璧に理解しようとすると止まります。
AIに聞いて「次の一歩」を作り、まず動かしてから理解を積み上げればOKです。

### おすすめWeb AI

| AI | 特徴 | URL |
|---|---|---|
| **ChatGPT** | 汎用性が高い、日本語も自然 | https://chat.openai.com |
| **Claude** | 長いコードの解説が得意、論理的 | https://claude.ai |
| **Gemini** | Google製、比較/要約が得意 | https://gemini.google.com |

### 質問の仕方（コピペ用テンプレ）

```
【状況】
このカリキュラムの「〇〇」の章を進めています。

【やりたいこと】
（例）配列を使ってToDoの一覧を表示できるようにしたい

【今のコード】
（ここにJavaコードを貼る）

【わからないこと】
どこをどう直せばいいか、原因から教えてください

【制約（おすすめ）】
答えをそのまま書かずに、まずはヒントと確認質問をください
```

### 重要な心構え

- **1回で覚えようとしない**（どうせ覚えない）
- 繰り返し聞いた回数が、そのまま理解度になる
- 「また同じこと聞いてる」は**成長の証拠**

---

## Step 2：同じことを何回も聞く（超重要）

「これ前も聞いたな...」と思ったら、それは正常です。

- 10回聞いて1回身につくのが普通
- 覚えようとするより、繰り返す方が早い
- **恥ずかしがらずに何度でも聞く**

### 効果的な「繰り返し方」

1. **同じ質問を別の言い方で聞く**
   - 「`==` と `equals` の違いは？」
   - 「なぜ `String` 比較で `==` は危険なの？」
   - 「`equals` を使わないと何が起きる？」

2. **具体例を変えて聞く**
   - 「`String` の比較でバグる例を教えて」
   - 「`List` で同じ落とし穴が起きる例はある？」

3. **自分の言葉で説明してみる**
   - 「`equals` は『中身が同じか』を比較する、で合ってる？」

## Step 3：課題の「丸写し」を防ぐAIの使い方

AIを使える環境では、答えに一瞬で到達できます。
ただし「答えを写す」だけだと理解が積み上がりません。
そこで、次のような聞き方に寄せてください。

### よい聞き方（理解が深まる）

- 「この章の練習問題1の方針を、答えを出さずに3つヒントでください」
- 「このエラー文を日本語で説明して、次に確認すべき点を順に教えて」
- 「自分のコードをレビューして、バグになりそうな箇所だけ指摘して」
- 「同じ処理をメソッドに切り出したい。どこを切るべきか理由付きで」
- 「自分の理解が合っているか確認したい。クイズを5問出して」

### 避けたい聞き方（理解が薄い）

- 「練習問題1の答え全部ちょうだい」
- 「動くコードを全部書いて」

Web AIだけで完結させないでください。
**AIコーディングエージェント**は、プロジェクト全体を理解して支援してくれます。

### Web AIとの違い

| 項目 | Web AI | AIコーディングエージェント |
|---|---|---|
| コード参照 | 貼り付けたコードのみ | プロジェクト全体 |
| ファイル編集 | コピペが必要 | 直接編集 |
| 影響範囲 | わからない | 自動で把握 |
| 実行確認 | 手動 | 自動で実行可能 |

### GUI系（初心者向け）

| ツール | 料金 | 特徴 |
|---|---|---|
| **Cursor** | 有料（無料お試しあり） | VSCode互換、直感的 |
| **Antigravity** | 無料 | ブラウザで完結 |

### CLI系（慣れたら効率爆上がり）

| ツール | 料金 | 特徴 |
|---|---|---|
| **Claude Code** | 有料 | 最も「考え方」を学べる、おすすめ |
| **Gemini CLI** | 無料 | Google製、軽量 |

### AIコーディングエージェントへの質問例

```
このプロジェクトでSpring BootのController/Serviceを追加したいんだけど、
どのファイルに何を書けばいい？
既存のコードに合わせて実装して。
```

エージェントは既存コードを読んで、プロジェクトに合った実装を提案してくれます。

## Step 4：Gitを使う（AI時代こそ必須）

AIを使うなら**Git/GitHub**は必須です。
「まだ早い」と思わず、今日から使いましょう。

### なぜ必要か

1. **何が変わったか分かる**
   - AIが変更した箇所が一目瞭然
   - 「何をどう直したか」を追跡できる

2. **壊しても戻せる**
   - AIに変なコードを書かれても一瞬で戻せる
   - 恐れずに実験できる

3. **自分の理解不足が可視化される**
   - 差分を見て「これ何してるの？」と気づく
   - AIに質問するきっかけになる

### 最低限覚えるGitコマンド（例）

```bash
# 変更を保存
git add .
git commit -m "ボタンを追加"

# 変更を確認
git status
git diff

# 戻す
git restore ファイル名

# GitHubに送る
git push
```

### AIに聞けばOK

```
git commitって何してるの？
わかりやすく教えて
```

Gitの使い方もAIに聞けば大丈夫です。

## Step 5：めんどくささをAIに投げる（ただし理由も聞く）

学習を進めていると「めんどくさい」が増えます。
それは**成長の合図**です。

### よくある「めんどくさい」

- 毎回同じimport文を書くのがだるい
- Maven/Gradleのエラーが長くて読む気がしない
- 同じコードを何度もコピペしてる

### 対処法

1. **まず、使っているツールで解決できないか調べる**
   - Eclipse/IntelliJのショートカットやテンプレート
   - 自動import、コード整形、リファクタ機能
   - 依存関係の表示（Maven/Gradle）

2. **AIに聞く**
   ```text
   Eclipseで毎回同じコードを書いてしまう（例：Scanner、例外処理、出力）。
   自分の状況だと、どの機能/ショートカットで楽できますか？
   ```

3. **自分で作るのは最終手段**
   - ほとんどの「めんどくさい」は既存ツールで解決できる
   - 車輪の再発明をしない

## 効果的なAI活用の4つの習慣

AIを使いこなすために、以下を意識しましょう。

### 1. AIの出力をレビューする習慣

AIの回答を鵜呑みにせず、必ず確認します。

```text
このコードが何をしているか、1行ずつ説明して
（最後に「この章の学び」として要点を3つにまとめて）
```

自分で理解できるまで質問を重ねましょう。

### 2. 自分の言葉で説明するフェーズ

AIに教わったことを、自分の言葉で説明してみます。

```text
「例外は、問題が起きたことを呼び出し元に伝える仕組み」だと思う。
合ってる？違うなら反例も出して。
```

### 3. 意図的に壊して学ぶ

動くコードを意図的に壊して、何が起きるか観察します。

```java
// わざと例外を起こしてみる（例）
String s = null;
System.out.println(s.length()); // どうなる？
```

壊して、直して、を繰り返すと理解が深まります。

### 4. AIなしの基礎体力を維持

定期的にAIなしでコードを書く時間を作りましょう。

- 小さな関数を手で書いてみる
- エラーメッセージを自分で読んでみる
- 公式ドキュメントを読んでみる

皮肉ですが、**AIを最強に使える人はAIなしでも書ける人**です。

## このカリキュラムの使い方（AI前提）

### 各章の構成

1. **目標** - この章で何ができるようになるか
2. **概念の説明** - 基本的な考え方
3. **コード例** - 実際のコード
4. **AIに質問する（この章の例）** - その章のコンテキストで聞ける質問例
5. **ハンズオン** - 手を動かす演習
6. **よくあるエラー** - つまずきポイント
7. **チェックリスト** - 理解度確認

### 推奨される学習フロー

```
1. 章を読む（完全に理解しようとしない）
     ↓
2. コード例を動かす
     ↓
3. わからないところをAIに質問
     ↓
4. ハンズオンをやってみる
     ↓
5. エラーが出たらAIに聞く
     ↓
6. 次の章へ（戻っても全然OK）
```

### わからないときのテンプレート

```
【このカリキュラムの〇〇章を勉強しています】
【今こういうコードを書いています：（コード）】
【このエラーが出ました：（エラー文）】
【何が原因でどう直せばいい？】
【答えを丸ごとではなく、ヒントからお願いします】
```

```
使う → 聞く → 壊す → 直す → また使う
```

このサイクルを回している人が**一番早く伸びます**。

- 「覚えてからやる」は遠回り
- 「やりながら覚える」が最速
- AIは**最強の個人教師**

さあ、次の章から実際にJava学習を始めましょう。
わからないことがあったら、遠慮なくAIに聞いてください。
