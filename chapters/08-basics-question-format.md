# 質問フォーマット


ここでは課題の提出、フィードバック、質疑応答、そのほかカリキュラムに関連すること全般を扱っていきます。

## **修正の仕方**

フィードバックに対して修正した際は、該当のフィードバックコメントを引用し、どこをどう修正したのかをしっかり記述してください！

```
例
1.　if (name == "takeshi")となっていますがStringでは「==」を使用しないでください。
　→　if　(name.equals("takeshi"))に修正しました。

2. 不要な変数が残っているようです。
　→　コードの簡潔性を保つために、不要な変数を削除しました。

```

## **質問の仕方**

カリキュラムを進めていく中で、疑問に思ったことやわからないことは何でも質問してください！
質問する際には以下の点に注意しましょう。
「◯◯◯という機能を実現したいのですが、エラーが出てしまいどうやっても解消できません。どのようにすれば実現できるのでしょうか？」
このような質問だと起きた結果だけを報告しているので、回答する側からすると「どんなエラーが出たのか？」「どのようなコードを書いたのか？」「何を試してダメだったのか？」などの情報が完全に抜けています。
そのため、これらの情報を得るために何度か質問をやり取りすることになり、お互いに無駄な時間を過ごすことになってしまいます。

質問するスキルは問題を解決する上でとても重要です。
他人の時間も自分の時間も無駄にしないために、以下の質問フォーマットを活用して、質問するスキルを向上させましょう。

**質問フォーマット**

```
１.　タイトル
（内容が分かるように重要なキーワードを意識して
1行で簡潔に質問内容を要約する）

２.　目的
（最終的に何をしたいのか、何をするための質問なのか）

３.　ゴール
（何ができたら問題を解決できるか明確に整理する）

４.　わからないこと・問題発生箇所
（問題が発生した経緯、エラーメッセージを記述する、
不具合のスクリーンショットを添付しても良い）

５.　調べたこと・試したこと
（参考にしたサイトのURL、自分で調べたこと、検索ワード、
試したことをできるだけ詳しく箇条書きにする）

```

上記をすべて埋めるは理想ですが、最初は以下のような状態に陥ると思いますので、1 ~ 5すべて埋められなくても質問してください！
・何がエラーメッセージなのか分からない
・そもそも自分が何をしようとしているのかも実のところ分かっていない

それでは、これからよろしくお願いします！
