## 動画内で紹介されている主要なWebサービス技術のまとめ

### リクエストとレスポンス

リクエスト
　PC、スマホからサーバーに情報を要求すること

レスポンス
　リクエストに応えてサーバーからPC、スマホ側に情報を返すこと

クライアントサイド
　情報を要求する側

サーバーサイド
　情報を返す側

IPアドレス
　リクエストを送る際に必要なサーバーの住所
　[例]Fecebook「31.13.82.1」

ドメイン
　数字の羅列であるIPアドレスに人間がわかりやすいようにつけられた紐づけされたIPアドレスの別名
　31.13.82.1 ↔ [facebook.com](http://facebook.com/)
　名前解決をしている

名前解決
　リクエストを送る時、http://facebook.comだけではIPアドレスがわからないので、名前解決を行う

DNS(ドメインネームシステム)
　インターネット上のドメイン名を管理するシステム
　複数のDNSサーバーによって成り立っている
　サーバーにリクエストを送る前に名前解決をしている

ドメインを買ったら必ず行う設定
　ドメイン名とサーバーのIPアドレスを紐付ける
　※サーバーを借りるとIPアドレスを教えてもらえる

### サーバとOS

サーバー
　リクエストに対していつでもレスポンスを返すように設定されたコンピューター
　※みなさんのPCも設定すればサーバーにできます

サーバーとして動くために必要な設定2つ
　①IPアドレスを固定する
　②リクエストに対してレスポンスを返すようにOSとアプリケーションを設定する

OS
　キーボードタッチパッドなどから入力情報をアプリケーションに伝える最も基本的なソフトウェアのこと
　[例]Windows、iOS、Android

サーバーによく使われるOS Linux
　サーバーOSとして世界的に大きなシェアを占める
　Linuxをベースとした様々なバージョンが存在、無料で利用できる
　■代表的なLinuxベースOS[CentOS、ubuntu]

離れたところにあるサーバーを設定する方法
　遠隔から自分のPCでネット接続してCUIのコマンド操作で設定していく

CUI
　コマンドラインユーザーインターフェイスの略
　黒画面に文字だけのやつ
　エンジニアはCUIで作業することが多い

GUI
　グラフィカルユーザーインターフェイスの略
　みんなが使っているマウスや指で操作できるやつ

### Webサイトの仕組み

HTML
　ウェブページを作成するために開発された言語
　文書の構造を説明したり文書の中に画像や動画を配置できる
　HTMLはサイトの「見た目(デザイン)」とは無関係
　HTMLはあくまでサイトの文章、画像の構造を決めるためのもの

CSS
　ウェブページのスタイル(見た目)を指定するための言語
　文章構造とデザインを切り分けることができる

構造とデザイン(HTMLファイルとCSSファイル)を分けるメリット
　例えば、デザインを変更したい場合CSSファイルの一部を修正するだけで全てのHTMLデザインが変えられる
　デザインの修正などがあった時に1つのCSSファイルを直せば良いから楽

WebサイトでWebページを出す時にやること
　サーバーにHTMLとCSSファイルをアップロードする
　レスポンスでファイルを返せばブラウザに表示される
　ブラウザでWebページを表示するために必要なのは「HTML、CSS、(+Javascript)」だけで他の言語を返しても動かない

### サーバーサイドスクリプトとデータベース

サービスにするために必要なもの
　①データを保存する仕組み
　②サービスの仕様を反映したロジック

①データを保存する仕組み
　データベースが担当

②サービスの仕様を反映したロジック
　サーバーサイドスクリプトが担当
　(Java/Ruby/PHP)

データベース
　データを保存するシステム
　通常エクセルのような表形式で保存され、1つの表をテーブルと呼ぶ
　あくまでデータの保存ができる場所でしかない

SQL(エスキューエル)
　データベースを操作するための言語
　テーブルの読み出し、追加、更新、削除などの操作ができる

サーバーサイドスクリプト
　データベースはあくまでデータの保存ができる場所でしかないため、サービスの仕様にもとづいたロジックや実際に返すHTMLの情報をプログラムしておく必要がある
　そのためにサーバーサイドスクリプトは使用される
　Java/Ruby/PHPを始めとするサーバーでの処理を担うプログラム
　これらを用いてロジックを書くことで動的なアプリケーションを作成できる

サーバーサイドスクリプトがやってくれること①
　必要に応じてSQLを用いてデータベースから情報を取得する

サーバーサイドスクリプトがやってくれること②
　データベースから取得したデータを利用してHTML/CSSを生成する
　データベースから取得したデータを利用してHTML/CSSに当てはめるとも言える

## まとめ

・リクエストとレスポンスの連続でサービスは成り立つ
・サーバーは特定のリクエストに対してレスポンスを返す電源ネットつけっぱのコンピューター
・サーバーにはLinux系のOSが入っている
・レスポンスでHTMLとCSSを返すことでウェブページを表示することができる
・リクエストが送られる
　　↪サーバーサイドスクリプトが欲しい情報をデータベースから持ってくる
　　　　↪もってきたデータをもとにHTML/CSSに生成してそれを返す
