# Javaエンジニアとして現場に行くためにはどんな知識が必要？


この章では、Javaエンジニアとして開発現場に行くために、どの程度の知識が必要かを説明します。
答えは、前回の章で紹介された技術全てが含まれます。
そう言われると果てしなく聞こえるかもしれませんが、もちろん最初は全てを完全に理解する必要はありません。

皆さんが行く開発現場が必要とする技術は、現場ごとに異なります。
ですので、最初は全てを理解することよりも、必要に応じて学習する柔軟性が重要です。

まずは、Javaエンジニアとして開発現場に行く上で必須の知識を優先的に学んでいきます。
開発現場では、プロジェクトの性質や要件に応じて異なる技術が求められますが、基本的なJavaの文法やプログラムの基本構造、オブジェクト指向プログラミングの考え方は必須です。

このような基本的な知識を身につけつつ、具体的なプロジェクトに必要な技術を都度学習していくことが、Javaエンジニアとしての成長に繋がります。
開発現場では、柔軟性と学習意欲が重要な要素となりますので、焦らず着実にスキルを積み重ねていくことが大切です。

## Javaエンジニア必須知識

1. Javaの基本文法
2. データ構造とコレクション
3. オブジェクト指向プログラミング
4. ファイル入出力、例外処理
5. 基本的なデータベースの理解
6. API
7. フレームワーク
8. 単体テスト
9. コーディング規約

これらの基本的な知識を身につけることで、Javaエンジニアとしての基礎が固まります。

## 必須知識の学習を終えたら学びたい知識

1. HTML、CSS、JavaScriptの知識
2. クラウドプラットフォーム
3. アプリケーションデプロイ
4. Linuxの知識
